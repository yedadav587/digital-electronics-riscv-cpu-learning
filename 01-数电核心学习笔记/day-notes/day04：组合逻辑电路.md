Day04：组合逻辑电路（加法器/译码器/MUX）

参考教材：清华大学《数字电子技术基础》（阎石 主编，第6/7版均可，核心内容一致）

核心说明：组合逻辑电路是数字电路的基础模块，其核心特征是任意时刻的输出仅取决于该时刻的输入信号，与电路过去的状态无关，电路结构中无记忆元件（如触发器），仅由逻辑门（与、或、非、与非等）和导线构成，无反馈回路。本章重点掌握加法器、译码器、数据选择器（MUX）的工作原理、电路结构、逻辑功能及典型应用，同时理解组合逻辑电路的通用分析与设计方法，为后续时序电路学习奠定基础。

一、组合逻辑电路基础（前置必备）

1.1 核心特征（教材重点强调）

- 功能上：输出仅由当前输入决定，输入变化时输出立即响应，无“记忆”能力；

- 结构上：仅包含逻辑门，无触发器、寄存器等记忆元件，无反馈路径；

- 数学描述：输出逻辑函数可表示为 Y = F(A₁, A₂, ..., Aₙ)，其中A₁~Aₙ为输入变量，Y为输出变量，无时间变量依赖。

1.2 通用分析方法（步骤固定，教材核心流程）

分析目的：已知电路结构，求解其逻辑功能（即输出与输入的逻辑关系），步骤如下：

1. 写出逻辑表达式：从输入端开始，逐级推导每个门的输出，最终得到总输出与输入的逻辑表达式；

2. 化简逻辑表达式：采用代数化简法或卡诺图化简法，将表达式化为最简与或式（或其他便于分析的形式），减少冗余项；

3. 列出真值表：根据化简后的逻辑表达式，列出所有输入变量的组合（2ⁿ种，n为输入变量个数）及对应输出值；

4. 分析逻辑功能：结合真值表，总结电路的核心功能（如算术运算、信号译码、数据选择等），若功能复杂可简要描述其应用场景。

1.3 通用设计方法（教材重点，与分析反向）

设计目的：已知逻辑功能需求，设计出满足需求的组合逻辑电路，步骤如下：

1. 逻辑抽象：将实际功能需求转化为数学模型，确定输入变量（原因）和输出变量（结果），定义逻辑状态（通常用“1”表示有效状态，“0”表示无效状态）；

2. 列出真值表：根据逻辑抽象的结果，列出所有输入组合及对应输出值，确保无遗漏、无错误；

3. 写出逻辑表达式：根据真值表，采用最小项之和法或最大项之积法，写出输出逻辑表达式；

4. 化简与变换：将表达式化简为最简形式，若需用特定门电路（如与非门、或非门）实现，需将表达式变换为对应形式；

5. 画出逻辑电路图：根据化简后的表达式，选用合适的逻辑门，画出电路连接图，确保连线正确、无冗余；

6. 验证：检查电路是否满足真值表要求，排查可能的逻辑错误（如竞争-冒险现象）。

1.4 竞争-冒险现象（教材必考点，补充说明）

定义：多路输入信号电平同时变化时，因信号传输延迟不同，导致输出端出现短暂的不正确尖峰脉冲（毛刺）的现象，称为竞争-冒险。

- 竞争：门电路两个输入信号同时向相反逻辑电平跳变的现象；

- 冒险：因竞争导致输出端产生尖峰脉冲的现象（有竞争不一定有冒险，有冒险一定有竞争）；

- 消除方法：① 接入滤波电容（抑制尖峰脉冲）；② 增加冗余项（修改逻辑表达式，消除冒险）；③ 采用格雷码（避免输入信号同时跳变）。

二、加法器（算术运算电路核心，教材重点）

加法器是实现二进制数加法运算的组合逻辑电路，是算术运算器（ALU）的核心模块，分为1位加法器和多位加法器，核心逻辑是“逢二进一”。

2.1 1位加法器（基础单元）

二进制加法中，1位加法分为两种情况：不考虑低位进位（半加）和考虑低位进位（全加），对应两种加法器。

2.1.1 半加器（Half Adder）

- 定义：仅实现两个1位二进制数（A、B）相加，不考虑来自低位的进位输入，输出为和数（S）和向高位的进位输出（Cₒᵤₜ）；

- 逻辑功能：

  - 当A、B不同时，和数S=1；当A、B相同时，和数S=0（异或逻辑）；

  - 当A、B同时为1时，进位Cₒᵤₜ=1；否则Cₒᵤₜ=0（与逻辑）。

- 逻辑表达式（最简）：
       

  - 和数：S = A ⊕ B（异或）

  - 进位：Cₒᵤₜ = A · B（与）

- 电路结构：由1个异或门（XOR）和1个与门组成；

- 真值表（教材标准）：
        输入A输入B和数S进位Cₒᵤₜ0000011010101101

- 特点：结构简单，但无法处理低位进位，仅适用于二进制加法的最低位运算。

2.1.2 全加器（Full Adder）

- 定义：实现两个1位二进制数（A、B）和来自低位的进位输入（Cᵢₙ）相加，输出为和数（S）和向高位的进位输出（Cₒᵤₜ），是多位加法器的核心单元；

- 逻辑功能：三个输入（A、B、Cᵢₙ）相加，遵循“逢二进一”，和数S是三个输入的异或，进位Cₒᵤₜ是三个输入中任意两个为1时产生；

- 逻辑表达式（推导与化简，教材重点）：
        

  - 和数：S = A ⊕ B ⊕ Cᵢₙ（三个变量异或，奇数个1时输出1）

  - 进位：Cₒᵤₜ = (A · B) + (A · Cᵢₙ) + (B · Cᵢₙ)（化简后，也可写成 Cₒᵤₜ = (A ⊕ B) · Cᵢₙ + A · B）

- 电路结构：由2个半加器和1个或门组成（第一个半加器实现A、B半加，第二个半加器实现第一个半加器的和与Cᵢₙ半加，或门实现两个半加器的进位输出相加）；

- 真值表（教材标准）：
        低位进位Cᵢₙ输入A输入B和数S高位进位Cₒᵤₜ0000000110010100110110010101011100111111

- 特点：可处理低位进位，是多位加法器的基础，实际应用中常集成化为标准芯片（如74LS183，包含2个独立全加器）。

2.2 多位加法器（实际应用核心）

当需要实现两个n位二进制数相加时，需将n个全加器级联，构成多位加法器，根据进位方式不同，分为串行进位加法器和超前进位加法器（教材重点讲解两种）。

2.2.1 串行进位加法器（Serial Carry Adder）

- 结构：将n个全加器依次级联，低位全加器的进位输出（Cₒᵤₜ）连接到高位全加器的进位输入（Cᵢₙ），最低位的进位输入（Cᵢₙ）接地（Cᵢₙ=0），最高位的进位输出（Cₒᵤₜ）作为整个加法器的进位输出（表示是否溢出）；

- 工作原理：加法从最低位开始，依次向高位进行，高位的加法必须等待低位的进位信号产生后才能完成（即“串行进位”）；

- 举例（2位串行进位加法器）：
       

  - 输入：A₂A₁（2位被加数）、B₂B₁（2位加数）；

  - 结构：2个全加器（FA₁：最低位，FA₂：高位）；

  - 连接：FA₁的Cᵢₙ=0，FA₁的Cₒᵤₜ → FA₂的Cᵢₙ，FA₂的Cₒᵤₜ为总进位；

  - 输出：S₂S₁（2位和数）、Cₒᵤₜ（总进位）。

- 优点：结构简单、成本低，易于扩展（增加全加器数量即可实现更多位加法）；

- 缺点：进位延迟大（位数越多，延迟越大），因为高位加法依赖低位进位，不适用于高速运算场景，教材中仅作为基础结构讲解。

2.2.2 超前进位加法器（Carry Look-Ahead Adder，CLA）

为解决串行进位加法器的进位延迟问题，教材重点介绍超前进位加法器，其核心是“提前计算出每一位的进位信号，无需等待低位进位”。

- 核心思想：通过输入变量（Aᵢ、Bᵢ）直接计算出每一位的进位Cᵢ₊₁，无需依赖低位进位Cᵢ，从而消除进位延迟；

- 关键定义（教材核心公式）：
        

  - 进位产生项：Gᵢ = Aᵢ · Bᵢ（当Aᵢ和Bᵢ均为1时，无论低位是否有进位，都会产生向高位的进位，即“主动进位”）；

  - 进位传递项：Pᵢ = Aᵢ ⊕ Bᵢ（当Aᵢ和Bᵢ不同时，低位的进位会传递到高位，即“被动进位”）；

  - 超前进位公式：Cᵢ₊₁ = Gᵢ + Pᵢ · Cᵢ（第i+1位的进位 = 第i位的进位产生项 + 第i位的进位传递项 × 第i位的进位）。

- 逻辑表达式推导（以4位超前进位加法器为例，教材标准）：
        

  - C₁ = G₀ + P₀ · C₀（C₀为最低位进位，通常为0）

  - C₂ = G₁ + P₁ · C₁ = G₁ + P₁ · G₀ + P₁ · P₀ · C₀

  - C₃ = G₂ + P₂ · C₂ = G₂ + P₂ · G₁ + P₂ · P₁ · G₀ + P₂ · P₁ · P₀ · C₀

  - C₄ = G₃ + P₃ · C₃ = G₃ + P₃ · G₂ + P₃ · P₂ · G₁ + P₃ · P₂ · P₁ · G₀ + P₃ · P₂ · P₁ · P₀ · C₀

  - 和数：Sᵢ = Pᵢ ⊕ Cᵢ（与全加器和数公式一致）

- 优点：进位延迟小，运算速度快，是高速数字系统（如计算机CPU）中常用的加法器结构；

- 缺点：结构复杂，位数越多，超前进位逻辑越繁琐（公式中项数越多），因此实际应用中常采用“分组超前进位”（如4位一组，组内超前进位，组间串行进位），平衡速度和复杂度；

- 典型芯片：74LS283（4位超前进位加法器），可直接用于4位二进制数加法，也可级联实现更多位加法。

2.3 加法器的典型应用（教材实例）

1. 二进制数加法运算：直接使用多位加法器实现两个n位二进制数的相加，如74LS283实现4位二进制加法；

2. 二进制数减法运算：利用“补码减法”，将减法转化为加法（A - B = A + [B]补），需增加求补电路（如反相器+1电路）；

3. BCD码加法运算：BCD码（8421码）加法后，若结果大于9（即1010~1111），需加6（0110）进行校正，因此BCD加法器 = 4位超前进位加法器 + 校正电路；

4. 数值比较器辅助：结合加法器实现两个数的大小比较（如A - B > 0，则A > B，通过进位输出判断）。

三、译码器（编码的逆运算，教材重点）

定义：将输入的二进制代码（n位）翻译成对应的输出信号（2ⁿ个），即“一个输入代码对应一个输出有效信号”，核心是“翻译”二进制代码的含义，分为二进制译码器、二-十进制译码器（BCD译码器）和显示译码器三类，教材重点讲解前两类及显示译码器的应用。

核心特征：输入为n位二进制代码，输出为2ⁿ个信号，任意时刻仅一个输出信号有效（高电平或低电平有效，由芯片决定），无输入代码时，所有输出均无效。

3.1 二进制译码器（最基础类型）

输入为n位二进制代码，输出为2ⁿ个独立的信号，每个输出对应一个输入代码的最小项，教材重点讲解2-4译码器、3-8译码器（最常用）。

3.1.1 3-8译码器（教材核心实例，如74LS138）

- 输入：3位二进制代码（A₂、A₁、A₀），A₂为最高位，A₀为最低位；

- 输出：8个信号（Y₀~Y₇），默认低电平有效（即输入代码对应哪一路，哪一路输出0，其余输出1），也可通过控制端改变有效电平；

- 控制端（使能端，教材重点强调）：74LS138有3个使能端（G₁、G₂A、G₂B），仅当 G₁=1、G₂A=0、G₂B=0 时，译码器工作；否则所有输出均为高电平（无效状态），使能端的作用是实现译码器的扩展；

- 逻辑表达式（低电平有效，教材标准）：
        注：符号“'”表示非运算，低电平有效时，输出表达式带非号。

  - Y₀' = (A₂'A₁'A₀')' （输入A₂A₁A₀=000时，Y₀=0，其余输出1）

  - Y₁' = (A₂'A₁'A₀)' （输入001时，Y₁=0）

  - Y₂' = (A₂'A₁A₀')' （输入010时，Y₂=0）

  - Y₃' = (A₂'A₁A₀)' （输入011时，Y₃=0）

  - Y₄' = (A₂A₁'A₀')' （输入100时，Y₄=0）

  - Y₅' = (A₂A₁'A₀)' （输入101时，Y₅=0）

  - Y₆' = (A₂A₁A₀')' （输入110时，Y₆=0）

  - Y₇' = (A₂A₁A₀)' （输入111时，Y₇=0）

- 真值表（74LS138，低电平有效，使能端有效）：
        使能端G₁使能端G₂A使能端G₂B输入A₂输入A₁输入A₀Y₇Y₆Y₅Y₄Y₃Y₂Y₁Y₀10000011111110100001111111011000101111101110001111110111100100111011111001011101111110011010111111100111011111110×××××11111111注：“×”表示任意电平（0或1），使能端无效时，所有输出均为高电平。

- 电路结构：由非门、与非门组成（输入先经非门产生反变量，再与原变量组合，通过与非门输出）；

- 扩展方法（教材重点）：利用使能端实现译码器位数扩展，例如用2片3-8译码器（74LS138）扩展为4-16译码器：
       

  - 将4位输入代码的最高位A₃作为扩展控制端，连接到第一片的G₁和第二片的G₂A、G₂B；

  - 4位输入的低3位A₂A₁A₀同时连接到两片的输入端；

  - 当A₃=0时，第一片工作，第二片无效，输出Y₀~Y₇；当A₃=1时，第二片工作，第一片无效，输出Y₈~Y₁₅，实现4-16译码。

3.2 二-十进制译码器（BCD译码器）

- 定义：输入为4位BCD码（8421码，0000~1001），输出为10个信号（Y₀~Y₉），对应十进制数0~9，输入超出BCD码范围（1010~1111）时，所有输出均无效；

- 核心区别：二进制译码器输入为n位，输出2ⁿ个；BCD译码器输入4位，输出10个（仅使用4位二进制代码的前10种组合）；

- 典型芯片：74LS42（BCD译码器，低电平有效）；

- 逻辑表达式（以74LS42为例，低电平有效）：
        

  - Y₀' = (A₃'A₂'A₁'A₀')' （输入0000，Y₀=0）

  - Y₁' = (A₃'A₂'A₁A₀')' （输入0001，Y₁=0）

  - ...（依次类推）

  - Y₉' = (A₃A₂'A₁'A₀')' （输入1001，Y₉=0）

  - 输入1010~1111时，所有Yᵢ'=1（无效）

- 应用：主要用于BCD码显示系统（如数字万用表、电子钟），将BCD码翻译成对应十进制数字的显示信号。

3.3 显示译码器（教材重点应用）

显示译码器是将BCD码或二进制代码翻译成数码管显示驱动信号的译码器，核心是“匹配数码管的显示逻辑”，分为共阴极和共阳极两种（对应不同类型数码管）。

- 数码管基础（教材补充）：
       

  - 七段数码管（最常用）：由7个发光二极管（LED）组成，分别对应a~g段，加上小数点dp，可显示0~9、A~F等字符；

  - 共阴极数码管：所有LED的阴极连接在一起（接地），阳极接高电平时LED点亮（高电平有效）；

  - 共阳极数码管：所有LED的阳极连接在一起（接电源），阴极接低电平时LED点亮（低电平有效）。

- 典型芯片：74LS48（共阴极七段显示译码器，高电平有效）、74LS47（共阳极七段显示译码器，低电平有效）；

- 74LS48的核心功能（教材重点）：
        

  - 输入：4位BCD码（A₃A₂A₁A₀）；

  - 输出：a~g（7段驱动信号，高电平有效），dp（小数点驱动，高电平有效）；

  - 控制端（辅助功能）：
            

    - LT（灯测试端）：LT=0时，所有段（a~g）点亮，用于测试数码管是否正常；

    - RBI（灭零输入端）：RBI=0且输入为0000时，数码管熄灭（灭零），用于多位显示时消除前导零；

    - BI/RBO（灭灯输入/灭零输出）：BI=0时，所有段熄灭；RBO为灭零输出，当灭零成功时输出0。

  - 真值表（核心部分，教材标准）：
            LTRBIA₃A₂A₁A₀abcdefg显示字符0×××××11111118（全亮）1000000000000灭零11000011111100110001011000011100101101101211100111110119

- 应用：数字显示系统（电子钟、万用表、计数器显示等），将BCD码通过显示译码器驱动数码管，实现十进制数字显示。

3.4 译码器的典型应用（教材重点）

1. 地址译码：在存储器系统中，将CPU输出的地址代码翻译成对应存储单元的选通信号，选中指定存储单元（如3-8译码器用于8个存储单元的选通）；

2. 显示驱动：通过显示译码器驱动数码管，实现数字显示（如电子钟中，将BCD码翻译成数码管显示信号）；

3. 组合逻辑电路设计：利用译码器输出为最小项的特点，结合或门（高电平有效）或与非门（低电平有效），实现任意组合逻辑函数（无需化简，直接将逻辑函数的最小项对应的译码器输出端连接到或门/与非门）；

4. 数据分配器：将一个输入数据分配到多个输出端中的一个，由译码器的输入代码控制分配方向（如3-8译码器可作为1-8数据分配器）。

四、数据选择器（MUX，Multiplexer）

定义：根据输入的地址代码，从多个输入数据中选择一个，输出到唯一的输出端，核心是“选通”，相当于一个可控制的单刀多掷开关，输入数据位数与输出数据位数一致，地址代码位数n与输入数据个数N满足 N = 2ⁿ（教材核心定义）。

核心特征：有N个数据输入端（D₀~Dₙ₋₁）、n个地址输入端（A₀~Aₙ₋₁）、1个输出端（Y），部分芯片有使能端（用于扩展），地址代码决定选择哪一路数据输出。

4.1 常用数据选择器（教材重点讲解）

按输入数据个数分类，常用的有2选1、4选1、8选1、16选1，教材重点讲解4选1和8选1，结合典型芯片说明。

4.1.1 2选1数据选择器（基础单元）

- 结构：2个数据输入端（D₀、D₁）、1个地址输入端（A₀）、1个输出端（Y）；

- 工作原理：由地址端A₀控制选通：
        

  - 当A₀=0时，选择D₀输出，Y=D₀；

  - 当A₀=1时，选择D₁输出，Y=D₁。

- 逻辑表达式：Y = A₀'D₀ + A₀D₁（化简后，本质是地址变量与数据变量的与或组合）；

- 电路结构：由2个与门、1个或门、1个非门组成（非门产生地址反变量，与门实现地址与数据的组合，或门汇总输出）；

- 真值表：
       地址A₀数据D₀数据D₁输出Y00×001×11×001×11注：“×”表示任意数据（不影响输出）。
